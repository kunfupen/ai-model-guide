## When to reach for GPT-5.5

GPT-5.5 is OpenAI's latest flagship — described as its "smartest and most intuitive
to use model yet" and the successor to the GPT-5 line. It's a strong all-rounder for
writing and debugging code, online research, data analysis, and agentic workflows that
move across tools until a task is finished.

- Hard reasoning and multi-step coding where answer quality matters more than cost.
- Agentic loops that plan, call tools, and recover from errors over many turns —
  GPT-5.5 powers OpenAI's Codex agents.
- High-stakes prompts where hallucination is costly: GPT-5.5 Instant produced ~52.5%
  fewer hallucinated claims than GPT-5.3 Instant on internal medicine/law/finance evals.

For high-throughput or latency-sensitive paths, the lighter GPT-5.5 Instant / mini
variants are the better fit; GPT-5.5 Instant is now ChatGPT's default for all users.

## Prompt patterns that work well

- **Trust it to think.** GPT-5.5 does its best work when you describe the goal and
  constraints and let it plan, rather than over-specifying the reasoning path.
- **Use developer messages for invariants.** Put hard rules in the developer/system
  message, not the user turn.
- **Lean on its memory tools.** It can refer back to past conversations and connected
  data for more personalized answers when you enable those tools.

## Tool use and agents

GPT-5.5 supports parallel tool calls and structured outputs natively. For agent loops,
prefer a small, well-named set of tools over a sprawling toolbox — selection quality
drops with too many similar-purpose tools.
