## When to reach for Kimi K2

Kimi K2 is Moonshot AI's flagship open-weights model — a 1T-parameter mixture-of-experts
(roughly 32B active per token) released as a permissive open release. It's a serious
alternative to closed-frontier models for teams that want strong coding and agentic
capability without sending tokens to a US-hosted API.

- Open-weights deployments where you can't or won't use a US API provider.
- Code generation and agentic loops where Moonshot's tool-use post-training shines.
- Cost-sensitive workloads — Moonshot's hosted pricing is well below the closed
  frontier, and self-hosted is cheaper still.

## Prompt patterns that work well

- **Be explicit about tools.** K2 was specifically post-trained on agentic patterns;
  it does best when given a small, well-documented set of tools and clear goals.
- **Use structured output for downstream parsing.** Asking for JSON or a tagged
  format works reliably.
- **Don't paste in the kitchen sink.** With 128k context it's tempting, but K2's
  attention quality holds up better when the prompt is curated.

## Hosted vs self-hosted

You can use K2 three ways:

1. **Moonshot's hosted API** — easiest path, OpenAI-compatible endpoints.
2. **Inference providers** (Together, Fireworks, Groq, etc.) — comparable pricing,
   different latency profiles.
3. **Self-hosted** — the MoE architecture means inference is heavier than a dense
   30B-class model; budget GPU memory accordingly.
