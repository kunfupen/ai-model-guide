## When to reach for Opus 4.8

Opus 4.8 is Anthropic's current frontier model and the successor to Opus 4.7.
It's the one to reach for when the task is genuinely hard: long-horizon agentic
work that calls tools across many steps, multi-file refactors, and reasoning that
has to hold together over a large context.

- Long-running coding agents that need to plan, edit, run tests, and self-correct.
- Tasks where you want the model to flag its own uncertainty rather than bluff —
  Opus 4.8 is notably more honest about its progress than 4.7.
- Work that benefits from the full 1M-token context window (available by default on
  the API, Amazon Bedrock, and Vertex AI).

## What's new vs. 4.7

- **Sharper judgement and honesty.** Early testers report it's more likely to flag
  uncertainties and less likely to make unsupported claims.
- **Stronger agentic coding.** It's roughly four times less likely than Opus 4.7 to
  let flaws in code it wrote pass unremarked, and posts the strongest end-to-end
  scores on Anthropic's internal Super-Agent benchmark.
- **Same price as 4.7.** $5 / 1M input and $25 / 1M output — with up to 90% savings
  via prompt caching and 50% via batch processing.
- **Faster fast mode and Dynamic workflows.** Opus 4.8 fast mode is ~2.5× quicker,
  and a new Dynamic workflows feature (research preview) lets Claude take on bigger
  tasks in Claude Code.

## Tradeoffs

- **Latency.** Deliberate by design — not the model for autocomplete or ultra-low-latency
  UX. Reach for Haiku 4.5 there.
- **Cost.** Still the priciest Anthropic tier per token; Sonnet 4.6 is the better
  default for the everyday 80%.
